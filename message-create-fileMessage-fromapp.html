Message: create / fileMessage (app -> client)

Back to overview

Confirm that the file message has been stored

This message SHALL be sent by the app once the message has been stored in the app's database.

After this message has been sent, the app MUST continue by uploading the blob to the blob server and the message to the chat server. By doing so, there MUST be continuous updates of the message with the returned messageId in form of update/messages towards the client.

The client receiving this message MUST wake up the app recurrently until the message with the returned messageId has been updated to the state sent.
