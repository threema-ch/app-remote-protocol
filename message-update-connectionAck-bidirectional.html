Message: update / connectionAck (bidirectional)

Back to overview

Acknowledges one or more chunks.

This message MUST be sent in case the remote peer requested a chunk acknowledgement via a request/connectionAck message.

If no chunk acknowledgement has been requested, this message SHOULD be sent after 10 seconds have elapsed once a chunk has been received.

Args

None

Data

Field Description Type Optional Nullable
sequenceNumber

The Sequence Number of the last chunk received during the connection

 Integer (u32) False False
