Message: request / keyPersisted (client -> app)

Back to overview

Mark the permanent key as established and the session as persistent.

The client SHALL send this message once it has stored the session persistently.

The app MUST defer storing a persistent session until this message has been received. This avoids persisting a session that has never been established, and thus can never be restored.
